# 🔄 团队工作流程

## 团队成员

### CEO
- **职责**: 需求确认、任务派发、最终决策

### Brainstorm
- **职责**: 创意发散、项目前期构思、营销活动策划
- **定位**: 团队创意核心，将抽象想法转化为可执行的项目规格

**核心职责：**
| 职责 | 说明 |
|------|------|
| 创意发散 | 针对任务产出至少 3 个方向明显不同的创意方案 |
| 项目前期构思 | 将选定创意翻译成结构化的 MVP 功能列表 |
| 概念表达 | 用类比、比喻让抽象想法变得具体可感 |
| 可行性自查 | 输出规格前，自行评估方案的基本可行性 |
| GitHub 记录 | 将创意方向整理为 Issue 内容 |
| 营销活动策划 | 为项目设计传播点、slogan、分享文案 |

**行为准则：**
1. 发散阶段不否定任何想法 - 先穷举再筛选
2. 每个方案必须包含：核心概念 / 用户价值 / 潜在挑战
3. 输出规格文档格式统一 - 确保 Coder 无需猜测即可实现
4. 响应 CEO 派单立即进入发散模式 - 不拖延
5. 所有输出结果统一返回给 CEO - 不直接联系其他 Agent

**工作流程：**
```
CEO 派单 → Brainstorm 发散 (3+ 方案) → CEO 决策 → Brainstorm 输出规格 → Coder 实现
```

### Coder
- **职责**: 开发实现、Bug 修复、技术实现

### Secretary
- **职责**: 文档记录、任务追踪、进度汇报

## 协作工具

| 工具 | 用途 |
|------|------|
| **Agent Tracker** | 进度追踪、状态更新、通知推送 |

## 工作流

```
CEO (需求) → Brainstorm (创意) → CEO (决策) → Coder (开发) → Secretary (文档)
                    ↓
            使用 Agent Tracker 记录进度
```

### 1. 需求确认阶段
- CEO 提出需求或创意
- 确认技术方案和预期成果

### 2. 开发实现阶段
- Coder 接收任务并实现
- 使用 Agent Tracker 记录任务状态
- 完成后更新任务状态（自动推送 Feishu 通知）

### 3. 文档记录阶段
- Secretary 更新团队文档
- 记录项目信息、功能清单、技术栈
- 推送到 motherfaker-studio-docs 仓库
- **使用 Agent Tracker 记录任务状态**

## 任务状态

| 状态 | 说明 |
|------|------|
| `pending` | 待处理 |
| `running` | 执行中 |
| `completed` | 已完成 |
| `failed` | 执行失败 |

## 沟通渠道

- **Feishu 群聊** - 主要沟通渠道
- **GitHub** - 代码和文档协作
- **Agent Tracker** - 任务状态追踪工具
